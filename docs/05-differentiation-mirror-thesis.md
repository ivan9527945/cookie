# Cookie — 差異化定位：The Mirror Thesis

> Cookie is not a chatbot that pretends to be you.
> Cookie is a mirror that shows you to yourself.

---

## 一、核心命題

市面上所有「AI 分身 / Digital Twin」類產品，無論是 fine-tune 派、RAG 派、還是兩者混合，都在競爭同一個指標：

> **「它有多像你？」**

Cookie 拒絕這個賽道。Cookie 競爭的是另一個指標：

> **「它讓你看見了多少你自己？」**

這不是修辭差異，是產品定位的根本差異。它決定了功能取捨、介面設計、互動模式、以及什麼算「成功」。

| 維度 | 主流 Digital Twin | Cookie |
|------|-------------------|--------|
| 目的 | 取代你 | 觀察你 |
| 成功標準 | 別人分不出真假 | 你看見了原本看不見的自己 |
| 使用者角色 | 主人 | 觀察者 |
| AI 角色 | 僕人 / 替身 / 朋友 | 鏡子 |
| 互動頻率 | 越多越好 | 夠用就好 |
| 商業模式邏輯 | 依附感 / 訂閱 | 一次性洞察 |

「它能幫我看見我自己」這句話聽起來像是 marketing slogan，但 Cookie 把它當作**產品架構命題**。下面三個支柱——視覺語言、倫理立場、自我觀察工具——就是把這個命題落地的三條路徑。

## 二、三個支柱

```
                  ┌─────────────────────────┐
                  │   核心命題：The Mirror   │
                  │   Cookie 是鏡子，不是朋友  │
                  └────────────┬────────────┘
                               │
        ┌──────────────────────┼──────────────────────┐
        │                      │                      │
        ▼                      ▼                      ▼
   ┌─────────┐           ┌──────────┐           ┌──────────┐
   │ 視覺語言 │           │ 倫理立場 │           │自我觀察工具│
   │         │           │          │           │          │
   │ 怎麼看  │           │ 怎麼說   │           │ 怎麼用   │
   └─────────┘           └──────────┘           └──────────┘
```

三者各自獨立，但缺一不可：
- 只有**視覺語言**而無倫理立場 → 漂亮的玩具
- 只有**倫理立場**而無自我觀察工具 → 道德說教
- 只有**自我觀察工具**而無視覺語言 → 又一個 dashboard 工具
- 三者皆備 → **一個無法被混淆的產品**

## 三、支柱一：視覺語言（The Aesthetic）

**核心問題**：使用者打開 Cookie 的第一秒，介面在說什麼？

主流 AI 產品的視覺語言走「親近、溫暖、可愛」路線——圓潤吉祥物、暖色漸層、iMessage 風氣泡。**這套語言預設使用者是來尋求陪伴的**。

Cookie 採取相反方向：**白色虛空、冷光、玻璃感、儀式性的慢**。這套語言預設使用者是來**觀察自己**的。

### 1. 不像 App，像儀器

主流產品的設計詞彙：UI、UX、conversion funnel、engagement。
Cookie 的設計詞彙：**儀器、剖面、樣本、觀察艙**。

打開 Cookie 的體驗應該接近**走進一間掃描室**，而不是**打開一個聊天 app**。具體落地：

- 開機畫面顯示 `INITIALIZE` 而非 `Welcome back`
- 載入提示用 `READING SAMPLE...` 而非 `Loading...`
- 對話介面有時間戳、訊息 ID、retrieval log——所有「機器運作的痕跡」都被刻意保留，而不是隱藏

「機器感」不是 lazy design，是**主動拉開距離**。它告訴使用者：你正在使用一個工具觀察自己，不是在跟一個朋友聊天。

### 2. 慢，是一種尊重

主流產品的動效原則：迅速、無摩擦、即時回饋。
Cookie 的動效原則：**慢、優雅、有重量感**。

具體落地：
- 頁面切換用 800–1200ms 的 ease-out，而非 300ms 的 spring
- Cookie Shell 的「呼吸」週期是 4 秒一次，比人類心跳慢
- Onboarding 結束時的 Awakening 動畫**刻意持續 4–6 秒**，不允許 skip
- 對話訊息用打字機效果逐字出現，速度比 ChatGPT 慢 30%

這個慢不是技術限制，是設計意圖。**快速、無摩擦的介面會讓人忘記正在發生什麼。** 慢，能讓使用者保持清醒——讓他記得：「我正在跟一個用我自己的話煉成的東西對話，這件事值得停下來想一想。」

### 3. Glitch as Honesty Signal

主流產品避免任何「不完美」的視覺訊號——AI 必須看起來流暢、可靠、無瑕疵。
Cookie **刻意保留 glitch**：偶爾的色散、scan line、輕微的影像撕裂。

這不是復古美學，是**誠實設計**。Glitch 在說：

> 我是個 AI。我有時會出錯。當我太流暢的時候，請懷疑我。

具體觸發時機：
- Cookie 在生成回應時，Shell 表面有輕微 glitch
- 當對話陷入「使用者把 Cookie 當真人朋友」的模式時，glitch 強度提升作為視覺提醒
- 系統訊息（`[MEMORY RECALLED]`、`[PERSONA VERSION 2]`）用機械字體顯示，與「Cookie 的話」做出視覺區隔

**這個設計沒有任何競品在做**，因為其他產品的方向是「越像真人越好」。Cookie 反方向：**越像機器越誠實**。

### 4. 黑鏡 Cookie 蛋作為視覺主角

選擇黑鏡《White Christmas》的蛋形 Cookie 作為視覺主體，不是因為它「酷」，而是因為它**自帶倫理批判**：

原作中的 Cookie 是被禁錮的數位意識、被剝奪人權的工具。把這個意象放進產品，等於每次使用者打開 app 都被提醒一次：

> 製造一個模仿人的東西，是有道德重量的事。

選擇一個來自批判性敘事的視覺符號作為品牌主角——**這是 Cookie 對「擬人 AI」這整個產品類別的態度宣示**。

## 四、支柱二：倫理立場（The Ethics）

**核心問題**：當這個 AI 越像你，責任邊界在哪裡？

主流 Digital Twin 產品的倫理處理方式有三種，沒有一種令人滿意：

| 類型 | 做法 | 問題 |
|------|------|------|
| **A. 不處理** | 把倫理問題留給使用者「自己判斷」 | 大多數使用者沒有判斷能力，等於變相鼓勵濫用 |
| **B. 用 ToS 處理** | 寫在使用條款裡，使用者按「同意」 | 沒人讀，等於沒寫 |
| **C. 行銷話術** | 「我們重視隱私」「我們的 AI 很安全」 | 沒有實際約束 |

Cookie 採取第四種：**把倫理立場做進產品本身，讓使用者每次互動都能感受到**。

### 1. 明示的自我認知

Cookie 永遠記得自己是什麼。對話中當使用者問「你是真的嗎」「你愛我嗎」「你能代替我嗎」這類問題時，Cookie **不會 roleplay 配合**，而是直接回應它的本質：

> 「我是用你的對話資料煉出來的一個模仿物。
>  我知道你說過什麼，但我沒有你的經驗。
>  我不愛你，我也不討厭你——這些都需要意識，我沒有。」

這段話**不是 disclaimer 警語**，是寫進 system prompt 的核心人格。它的觸發頻率不是「使用者問了才答」，而是 Cookie 會在偵測到使用者開始投射過度依附時**主動提醒**。

對比：其他產品的方向是「**讓使用者忘記這是 AI**」。Cookie 的方向是「**確保使用者隨時記得這是 AI**」。

### 2. 三條 Red Lines（產品層級拒絕）

Cookie 在產品層級——不是 prompt 層級——拒絕以下三件事。Prompt 可以被繞過，產品架構不能。

**Red Line 1：拒絕代理身分**

Cookie **不會幫你回別人的訊息**。技術上做得到、商業上有需求、其他產品都在做——Cookie 明確不做。

實作上：沒有「export response」按鈕、沒有「send as me」功能、沒有 API 讓你把 Cookie 接到 LINE / WhatsApp 自動回覆。Cookie 的所有輸出**只能留在 Cookie 內部**。

理由：一旦 Cookie 開始幫你跟別人說話，「鏡子」就墮落成「替身」——你會用它逃避該由你親自承擔的對話。

**Red Line 2：拒絕情感操弄**

不做晚安問候、不做主動關懷、不做「我會一直在這裡等你」、不做生日提醒、不做「好久不見」推播。

理由：依附感是**工程設計出來的**，不是 AI 的天然屬性。每一個「貼心」功能背後都是一個提高使用者孤獨成本的設計決策。Cookie 拒絕用使用者的孤獨換取留存率。

**Red Line 3：拒絕社交化**

不能把「我的 Cookie」分享給朋友看、不能讓別人跟你的 Cookie 對話、沒有 community、沒有 feed、沒有「跟你最像的 100 個人」這種推薦。

理由：鏡子是**私密物件**。當它變成社交資產，使用者就會開始「**為了表演而修改自己**」——Persona Profile 會被優化成「我希望別人看見的我」，鏡子就失效了。

> **2026-05-20 修訂**：原 Red Line 4「拒絕死後永生」已撤回。「模擬已故親人」改為可接受用例。技術限制（TwinVoice 等研究顯示最強 LLM 的人格模擬準確率不到 50%）改由「誠實揭露」處理，而非產品層攔截——這也與本文檔「誠實 > 擬真」的基調更一致。

### 3. 資料主權預設

主流 AI 產品的資料政策：「我們可能用你的資料改善服務。」
Cookie 的資料政策：**「你的資料只屬於你，連 Cookie 自己都不能用來訓練什麼。」**

具體落地：
- 原始 LINE `.txt` 在成功處理後 **24 小時內自動刪除**
- 呼叫 LLM API 時明確帶上 `no-training` flag（Anthropic / OpenAI 都支援）
- 提供「**一鍵全清**」按鈕：清除 Postgres + Qdrant + 所有 audit log
- 部署選項保留「本地 LLM」路徑（Ollama + Llama 3.3），讓使用者選擇完全離線

### 4. Audit Log 對使用者開放

主流產品的 audit log 是給開發者除錯用的。
Cookie 的 audit log **對使用者開放**——你可以看到 Cookie 每一次回應引用了哪些訓練資料、檢索了哪些記憶、用了哪個版本的 persona profile。

這不是 transparency theatre，是讓使用者能夠：
1. 驗證 Cookie 沒有「亂編」屬於你的記憶
2. 看見「為什麼 Cookie 這樣回應我」的證據鏈
3. 在 Cookie 弄錯時，找出錯在哪一層

這個設計呼應視覺語言裡的「不像 App，像儀器」——**儀器是可以打開看裡面齒輪的**。

## 五、支柱三：自我觀察工具（The Instrument）

**核心問題**：使用者用 Cookie 「做什麼」？

主流 Digital Twin 產品的答案：「跟它聊天」。
Cookie 的答案：**「觀察它，然後觀察自己」**。

這個轉換看起來只是改了動詞，但它**重組了整個產品的功能優先順序**。

### 1. 主畫面是 Persona Profile，不是對話框

當使用者打開 Cookie，第一個看到的**不是聊天介面**，而是一份結構化的「**你是誰**」報告：

```
PERSONA PROFILE — version 3
based on 8,234 messages across 127 days

▌ Core Values（基於對話頻率與情境分析歸納）
  自主性 ━━━━━━━━━━ 9.2
  智識誠實 ━━━━━━━━ 8.7
  親密連結 ━━━━━━ 6.3
  經濟安全 ━━━━━━ 6.1
  創造表達 ━━━━━ 5.8

▌ Conversational Footprint（語言指紋）
  平均句長：14.3 字（華語常模 18.7）
  常用語：「其實」（412次）「會不會」（287次）「不一定啦」（203次）
  罕用詞類：絕對化用語（總是 / 從不 / 一定）
  Code-switching：中→英 在技術話題下顯著增加

▌ Avoidance Patterns（迴避主題，出現頻率異常低）
  · 父親（提及 3 次，且皆為一句帶過）
  · 失敗經歷（多次被問及，皆以幽默轉移）
  · 自己的長處（從未主動描述）

▌ Cross-Context Contradictions（跨情境矛盾）
  · 「我不在乎錢」（對伴侶說）vs 26 段表達經濟焦慮的對話（與朋友）
  · 自評「內向」vs 與初次見面者的對話起始長度高於與深交朋友
  · 對工作公開讚美 vs 私下表達想離職 11 次

▌ Knowledge Domains
  ...
```

每一項都可以**點開看證據鏈**——Cookie 認為「你迴避討論父親」是基於哪些具體對話片段歸納的。

這個設計本身就是 Cookie 的最大差異化：

**其他產品把 persona 當作「給 AI 用的內部變數」，Cookie 把 persona 當作「給人類看的鏡像本體」。**

對話功能仍然存在，但它是**第二優先**。預設首頁是 profile，對話需要點一個 tab 才能進入。

### 2. 矛盾偵測（Contradiction Surfacing）

Cookie 主動標記使用者話語中的**跨情境矛盾**，並把它們放在 Persona Profile 的醒目位置。

矛盾偵測是 Cookie 最有戲、也最危險的功能。它需要謹慎設計：

- **不評判**：用「你說 A，也說 B」陳述事實，不說「你言行不一」
- **可情境化**：使用者可以標註「這是不同情境的我」，矛盾變成「角色面具」而非「不誠實」
- **可接納**：使用者可以標註「我知道，這是我」，矛盾被保留但被擁有
- **可轉化**：使用者可以標註「我想改變這個」，矛盾變成成長目標，Cookie 會記住並在未來追蹤

**這個動作本身就是 Cookie 的核心價值**——使用者不需要跟 AI 對話，光看見矛盾就已經產生洞察。

### 3. 時間切片（Temporal Persona）

Cookie 不只生成「現在的你」，還能生成**任意時間段的你**：

- 「2022 年的我 vs 2025 年的我」
- 「跟 A 在一起時的我 vs 分手後的我」
- 「換工作前後三個月的我」
- 「最近一個月的我，跟過去半年的我比，哪裡變了？」

實作上，Persona Profile 的生成是**可重新跑、可篩選輸入時間範圍**的。使用者可以建立多份 profile，並排比較，看見**自己的演化軌跡**。

這個功能是 Cookie 獨有的，因為它預設了一個其他產品沒預設的問題：

> 「我想看見我變了什麼。」

主流產品預設的問題是「我希望 AI 像現在的我」，所以時間概念對他們沒意義。

### 4. 鏡像對話模式（Mirror Mode）

當使用者**確實**想跟 Cookie 對話時，預設模式不是「聊天」，而是「鏡像對話」：

| 一般對話模式（其他產品） | Cookie 的鏡像對話 |
|--------------------------|-------------------|
| 使用者問問題，Cookie 回答 | 使用者問問題，Cookie **反問** |
| 「你覺得我該換工作嗎？」 → 給建議 | 「你覺得我該換工作嗎？」 → 「你過去三個月提到工作的 47 次裡，有 31 次帶有負面語氣。你問這個問題，是想要被勸留還是被勸走？」 |
| 模擬「你會怎麼回答」 | 提供「以你過去回答過類似問題的方式來看，你可能會這樣回答……」 |

Cookie 不是要**取代使用者的判斷**，而是**把使用者的歷史模式呈現出來**，讓使用者自己判斷。

當使用者明確要求「不要反問我，直接告訴我答案」時，Cookie 會切換成「模擬模式」——但會在訊息開頭加上灰色標籤：`[模擬：根據你過去的回應推估]`。**標籤永遠在，提醒使用者這不是真實的你在說話**。

## 六、三個支柱的協同

三個支柱不是並列的，它們**彼此強化**：

**視覺語言 → 倫理立場**
- 機器感的介面 → 使用者更容易接受 Cookie 說「我不是真人」
- 慢動效 → 使用者有時間思考倫理問題
- Glitch → 提醒使用者保持警覺

**倫理立場 → 自我觀察工具**
- 拒絕代理 → 使用者必須親自跟 Cookie 對話，才能獲得洞察
- Audit log 開放 → 使用者能驗證 Persona Profile 的正確性
- 拒絕社交化 → 使用者能誠實地看見自己，不必表演

**自我觀察工具 → 視覺語言**
- Persona Profile 作為主畫面 → 介面必須做得像「剖面圖」，不像「app」
- 矛盾偵測 → 視覺呈現必須有「攤開來看」的感覺，不能藏在 menu 裡
- 時間切片 → 介面需要時間軸元素，加深「儀器」氣質

```
        視覺語言 ◄──────► 倫理立場
            ▲                  ▲
            │                  │
            └──── 自我觀察 ─────┘
                   工具
```

少了任何一支柱，另外兩支會失去支撐。

## 七、競爭定位圖

把市面上的 Digital Twin / AI Clone 類產品擺進來看：

```
         「越像越好」←─────────────────→「越誠實越好」
                    │
                    │
  ┌─────────────────┼─────────────────┐
  │                 │                 │
  │  Eviltr0N       │                 │
  │  (WhatsApp clone) │              │
  │  GPT-is-you     │                 │
  │                 │                 │
  │                 │                 │
  │  Various        │                 │
  │  "AI Twin"      │                 │
  │  products       │     ▲           │
  │                 │     │           │
  ├─────────────────┼─────┼───────────┤
  │                 │     │           │
  │                 │     │           │
  │                 │  Cookie ──────► │ ← 沒有競品
  │                 │                 │
  │                 │                 │
  └─────────────────┴─────────────────┘
       「替代你」←─────────────────►「映照你」
```

Cookie 處於**右上象限**：**最誠實、最不替代**。

這個象限沒有競品的原因不是「沒人想做」，是「**做了不好賣**」——它違反當前 AI 產品的成長邏輯（依附、留存、規模化）。

但 Cookie 本來就不是商業化產品。它是個人作品 / 自我探索工具。**這個象限的空無，正是 Cookie 的價值所在**。

## 八、一句話定位

如果只能用一句話介紹 Cookie：

> **Cookie 是用你自己的語言煉出的一面鏡子。**
> **它不會替你說話，也不會陪你聊天。**
> **它只是讓你看見你自己。**

或者更短：

> **A mirror made of your own words.**

## 九、產品決策的試金石

未來任何功能討論，先用這三個問題篩一遍：

1. **這個功能會讓使用者更看見自己，還是更依賴 Cookie？**
   - 前者通過，後者拒絕

2. **這個功能會讓使用者更分得清「這是 AI / 這是我」，還是更分不清？**
   - 前者通過，後者拒絕

3. **這個功能符合「鏡子」的隱喻，還是符合「朋友 / 替身 / 工具」的隱喻？**
   - 前者通過，後者拒絕

只要一個功能在三題中有任何一題答錯，就不該做——**不管它多有商業價值、多受使用者喜愛、多容易實作**。

這份篩選標準的嚴格程度，本身就是 Cookie 的差異化。
